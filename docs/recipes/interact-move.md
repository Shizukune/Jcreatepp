# 触ったら動く

## 何ができる？

プレイヤーがアイテムを触ったときに、そのアイテムを少し動かします。

例として、スイッチを押すと足場が上がる、ボタンを押すと箱が横にずれる、といった動きに使えます。

## 使う場面

- 押すと開くドア
- 触ると上がる足場
- 説明パネルを押すと別の場所へ移動する仕掛け

## 必要なUnity / Cluster設定

- スクリプトを動かすアイテムに Scriptable Item を設定します。
- プレイヤーが触れる対象として使う場合は、Interactできる設定が必要になることがあります。
- アイテム自体を動かす場合は、Cluster側で移動可能なアイテムとして扱える設定が必要になる場合があります。

公式確認メモ: アイテムを動かす設定や制約はClusterの更新で変わる可能性があるため、公開前に公式ドキュメントも確認してください。

## Jcreate++で置くブロック

1. `触ったとき`
2. `位置を変える` または `指定位置へ移動`
3. 必要なら `メッセージを表示` や `状態を保存`

## 生成される考え方

「触ったとき」というイベントの中で、対象アイテムの位置を変更します。

コードの考え方としては、`onInteract` のようなイベントで処理を開始し、アイテムの位置を書き換える形です。

## 注意

- 動かしたい対象が「自分自身」なのか「別のアイテム」なのかを決めておきます。
- 別のアイテムを動かす場合は、World Item Reference のような参照設定が必要になることがあります。
- 何度も触ると移動量が積み重なる設計にするか、毎回同じ場所へ移動する設計にするかを先に決めると迷いにくいです。

## よくあるミス

- Scriptable Item が付いていない
- Interactできる対象になっていない
- 動かしたいアイテムの参照名が間違っている
- 移動量が大きすぎて、アイテムが見えない場所へ飛んでいる

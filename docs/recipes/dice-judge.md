# サイコロ判定

## 何ができる？

ランダムな数値を使って、当たり・はずれや分岐を作ります。

例として、触るたびにランダムで宝箱の中身を変える、確率で扉が開く、といった処理に使えます。

## 使う場面

- ランダム宝箱
- おみくじ
- 確率で成功するボタン
- サイコロの目によって動きが変わるギミック

## 必要なUnity / Cluster設定

- スクリプトを動かすアイテムに Scriptable Item を設定します。
- 結果に応じて別アイテムを動かす場合は、そのアイテム参照の設定が必要になる場合があります。
- 音や見た目を変える場合は、Audio Set や Material Set などの設定が必要になる場合があります。

公式確認メモ: ランダム値自体はスクリプト側で扱えますが、結果に応じた表示・音・移動はUnity側設定が必要になることがあります。

## Jcreate++で置くブロック

1. `触ったとき`
2. `AからBのランダムな数`
3. `もし 〜 なら`
4. 当たりの場合の処理
5. はずれの場合の処理

## 生成される考え方

イベントが起きたタイミングでランダムな数値を作り、その値を条件分岐に使います。

たとえば「1から6のランダムな数」を作り、「4以上なら当たり」と決めるような考え方です。

## 注意

- ランダムの範囲を決めるときは、最小値と最大値を間違えないようにします。
- 小数を使うのか、整数として使うのかを先に決めると分岐が読みやすくなります。
- 確率を厳密にしたい場合は、生成される数値の扱いを確認してください。

## よくあるミス

- 条件が常に真、または常に偽になっている
- ランダム値を作ったあと、分岐で別のランダム値をもう一度作ってしまう
- 当たり処理とはずれ処理が同じになっている
- 表示や音のUnity側設定を忘れている

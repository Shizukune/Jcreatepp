# クールダウンつきボタン

## 何ができる？

一度押したあと、一定時間たつまで再び押せないボタンを作ります。

連打防止、再使用までの待ち時間、一定時間ごとに使える仕掛けに向いています。

## 使う場面

- 連打できないスイッチ
- 一定時間ごとに使える回復ボタン
- 何秒か待つと再挑戦できるランダム判定
- 開閉に待ち時間があるドア

## 必要なUnity / Cluster設定

- スクリプトを動かすアイテムに Scriptable Item を設定します。
- ボタンとして触る場合は、Interactできる設定が必要になることがあります。
- 残り時間を見せたい場合は、表示用の仕組みを別途用意する必要があります。

公式確認メモ: 時間経過の扱いは、Cluster Scriptの更新処理や状態保存の仕様に関係します。公開前に実機で確認してください。

## Jcreate++で置くブロック

1. `触ったとき`
2. `もし クールダウン中ではない なら`
3. 実行したい処理
4. `クールダウンを開始する`
5. 必要なら `あと何秒か` を表示

## 生成される考え方

クールダウンの残り時間を `$.state` のような状態として持ちます。

毎フレーム、または時間更新の処理で残り時間を減らし、0秒になったら再び使えるようにします。

## 注意

- クールダウン中に押されたとき、何もしないのか、メッセージを出すのかを決めておきます。
- 時間を使う処理は、同じ場所でまとめて更新する設計にすると壊れにくくなります。
- 複数のボタンで同じクールダウン名を使うと、意図せず共有されることがあります。

## よくあるミス

- クールダウン開始を忘れて、何度でも押せる
- 残り時間を減らす処理がない
- クールダウン名を間違えて、別の処理と混ざる
- 0秒以下になったときの扱いを決めていない

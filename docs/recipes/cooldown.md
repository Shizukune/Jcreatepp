# クールダウンつきボタン

## 何ができる？

一度押したあと、一定時間たつまで再び押せないボタンを作ります。

連打防止、再使用までの待ち時間、一定時間ごとに使える仕掛けに向いています。

## 使う場面

- 連打できないスイッチ
- 一定時間ごとに使える回復ボタン
- 何秒か待つと再挑戦できるランダム判定
- 開閉に待ち時間があるドア

## 必要なUnity / Cluster設定

- スクリプトを動かすアイテムに Scriptable Item を設定します。
- ボタンとして触る場合は、Interactできる設定が必要になることがあります。
- 文字で残り時間を見せたい場合は、表示用サブノードを用意します。
  - 例: サブノード名 `Text`
  - 例: コンポーネント `TextView`
- アイテムを回転・移動する場合は、移動可能な設定が必要になることがあります。

## Jcreate++で置くブロック

### 一番シンプルな形

1. `インタラクト時`
2. `もし クールダウン action 中 ではない なら`
3. 中に `角度を x:0 y:0 z:90 ずつ変える`
4. 中に `クールダウン action を 3 秒で開始する`

### 残り時間も表示する形

1. `インタラクト時`
2. `もし クールダウン action 中 なら`
3. 中に `サブノード Text の TextView の文字を クールダウン action の残り秒数 にする`
4. `でなければ` 側に `クールダウン action を 3 秒で開始する`
5. `でなければ` 側に `角度を x:0 y:0 z:90 ずつ変える`

`action` はクールダウン名です。好きな名前でよいですが、同じ仕掛けでは同じ名前にそろえます。

## 生成される考え方

`action` という名前で、クールダウンの残り秒数を内部状態として覚えます。

`クールダウン action 中` が true の間は、ボタンを押しても本処理を実行しません。0秒になったら、また押せるようになります。

## 注意

- クールダウン中に押されたとき、何もしないのか、残り秒数を表示するのかを決めます。
- 残り秒数を表示する場合は、`文字変数 action` ではなく `クールダウン action の残り秒数` を使います。
- 複数のボタンで同じクールダウン名を使うと、意図せず共有されることがあります。

## よくあるミス

- `クールダウン action を 3 秒で開始する` を入れ忘れる
- 条件を逆にして、クールダウン中だけ本処理を実行してしまう
- 表示用サブノード名 `Text` がUnity側と違う
- Unity側が `Text View (Script)` なのに、Jcreate++側で `Text` を選んでいる

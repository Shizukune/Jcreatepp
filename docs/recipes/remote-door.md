# 遠隔スイッチで開閉するドア

## 何ができる？

スイッチをインタラクトすると、別アイテムのドアにメッセージを送り、ドアを開閉します。

もう一度スイッチを押すと、同じドアが元の位置へ戻ります。

## 使う場面

- 壁のボタンで開くドア
- 離れた場所の仕掛けを動かすスイッチ
- 複数のアイテムをメッセージで連動させる練習

## 必要なUnity / Cluster設定

スイッチ側:

- Scriptable Itemを付けます。
- インタラクトできる設定にします。
- World Item Reference List に、ドアを `door` という名前で登録します。

ドア側:

- Scriptable Itemを付けます。
- 位置をスクリプトから変えるため、移動できる設定にします。
- `メッセージ toggle を受け取ったとき` を使います。

## Jcreate++で置くブロック

### スイッチ側

1. `インタラクト時`
2. 中に `もし 条件 true になったら 指定アイテム door に メッセージ toggle を1回だけ送る`

`true` は「この処理を実行する」という意味の固定条件です。
World Item Reference List 側の名前と、Jcreate++側の `door` を必ずそろえます。

### ドア側

1. `メッセージ toggle を受け取ったとき`
2. 中に `もし フラグ open なら / でなければ`
3. `もし` 側に `位置を x:0 y:-2 z:0 だけ 1 秒かけて動かす`
4. そのあと `フラグ open を false にする`
5. `でなければ` 側に `位置を x:0 y:2 z:0 だけ 1 秒かけて動かす`
6. そのあと `フラグ open を true にする`

## 生成される考え方

スイッチ側は `door` という参照名のアイテムへ `toggle` メッセージを送ります。

ドア側は `toggle` を受け取ったら、自分の `open` フラグを見ます。

- `open` が true: 閉じる方向へ動かして、最後に false にする
- `open` が false: 開く方向へ動かして、最後に true にする

滑らか移動は、受信した瞬間に移動を開始し、内部の毎フレーム処理で少しずつ位置を変えます。

## 注意

- ドアの開く方向は、ワールド上の向きに合わせて `x / y / z` を調整してください。
- `位置を x:0 y:2 z:0 だけ 1 秒かけて動かす` は、現在位置から相対的に動きます。
- ドアを横に動かしたいなら、例として `x:2 y:0 z:0` にします。
- 回転ドアにしたい場合は、位置ではなく `角度を x:0 y:90 z:0 だけ 1 秒かけて回す` を使います。
- スイッチを連打すると動きが重なることがあります。必要ならクールダウンを足します。

## よくあるミス

- World Item Reference List の名前が `door` と一致していない
- スイッチ側のメッセージが `toggle` ではない
- ドア側に `メッセージ toggle を受け取ったとき` が置かれていない
- フラグ `open` の true / false が逆になっている
- ドアを動かせるUnity設定になっていない

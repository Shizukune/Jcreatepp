# 動かないときの確認リスト

Jcreate++で作ったスクリプトが動かないときは、コードより先に設定を確認すると早いことがあります。

## 1. Scriptable Itemは付いていますか？

スクリプトを動かしたいアイテムに Scriptable Item が設定されているか確認します。

別のアイテムに付いていると、期待した場所では動きません。

## 2. 触る系のギミックはInteractできますか？

`インタラクト時` の処理を使っている場合、アイテムがプレイヤーから触れる状態になっているか確認します。

- 触れる位置にあるか
- Colliderなどの設定が足りているか
- Interact対象として扱えるか

## 3. 参照名は合っていますか？

別のアイテム、音、マテリアルなどを使う場合、Unity側のリスト名とJcreate++側の指定が合っている必要があります。

大文字・小文字、全角・半角、余計な空白に注意してください。

## 4. 状態の名前はそろっていますか？

`$.state` のような状態を使う処理では、保存する名前と読む名前が一致している必要があります。

例:

- `cooldown` に保存したのに、読むときは `coolDown` になっている
- `doorOpen` と `isDoorOpen` を混ぜている

## 5. 条件分岐が逆になっていませんか？

「クールダウン中なら実行しない」「当たりなら開く」のような条件は、逆に書くと動いていないように見えます。

一度、条件の中身を声に出して読むと見つけやすいです。

## 6. ランダム値を確認できますか？

ランダム判定は、結果が見えないと失敗しているように感じます。

最初はメッセージ表示やログ代わりの処理を入れて、どの値が出ているか確認できるようにすると安心です。

## 7. 時間を使う処理は更新されていますか？

クールダウンやなめらかな移動は、時間経過で状態を変える必要があります。

残り時間を減らす処理や、移動を進める処理が入っているか確認してください。

## 8. まず小さく試しましたか？

大きな仕掛けを一気に作ると、どこで失敗したか見つけにくくなります。

おすすめの順番:

1. 触ったらメッセージを出す
2. 触ったら状態を変える
3. 状態に応じて動きを変える
4. 音や見た目を足す

1つずつ動かすと、原因をかなり絞れます。

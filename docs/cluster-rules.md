# Clusterルール早見表

Jcreate++で詰まりやすいCluster Scriptまわりのルールです。

## メッセージ

- メッセージ名は短くします。
- 値つきメッセージは `数値`、`文字`、`真偽値` から始めるのがおすすめです。
- 毎フレーム送信すると制限に当たりやすいので、`1回だけ` や `クールダウン` を使います。

## World Item Reference

指定アイテムへ送る場合は、Unity側で `World Item Reference List` が必要です。

例:

- Jcreate++側の名前: `door`
- Unity側の登録名: `door`

大文字小文字も同じにします。

## 衝突

`衝突したとき` を使うには、Collider / Physical Shape などの衝突判定設定が必要です。

`衝突相手` は、相手がアイテムまたはプレイヤーの場合に使えます。  
それ以外の相手では空になることがあります。

## 移動

位置や回転をスクリプトで変えるアイテムは、移動できる設定が必要です。

動かないときは、まずUnity側のアイテム設定を確認します。

## クールダウン

クールダウンは連打防止に使います。

例:

1. まだクールダウン中でなければ処理する。
2. 処理したらクールダウンを開始する。
3. 残り秒数が0になったら再び動く。

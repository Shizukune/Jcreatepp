# イベント

イベントは「いつ処理を始めるか」を決める入口です。

イベントブロックは基本的に一番上に置き、その中に動作や条件のブロックを入れます。

## 開始時

アイテムが出現したときに1回だけ実行します。

使いどころ:

- 初期位置を保存する
- 初期フラグを設定する
- 最初の表示文字を入れる

注意:

- プレイヤー情報はありません。
- 衝突相手や受信値は使えません。

## 毎フレーム

フレームごとに繰り返し実行します。

使いどころ:

- 回転し続ける
- 往復する
- 追跡する
- クールダウン時間を進める
- 範囲内プレイヤー検知を続ける

注意:

- 毎フレーム送信や毎フレーム音再生は重くなりやすいです。
- 1回だけ実行したい処理は `once` やクールダウンを使います。

## インタラクト時

プレイヤーがアイテムを触ったときに実行します。

使いどころ:

- ボタン
- スイッチ
- ガチャ
- 文字入力要求の入口
- プレイヤー速度変更

使える値:

- `プレイヤー`

## 衝突したとき

このアイテムが何かと衝突したときに実行します。

使いどころ:

- ゴール判定
- 当たり判定
- 衝突相手にメッセージを送る
- ぶつかった相手だけ反応させる

使える値:

- `衝突相手`

注意:

- Unity側でColliderやPhysical Shapeなどの設定が必要です。

## 持ったとき / 離したとき

プレイヤーがアイテムを持った、または離したときに実行します。

使いどころ:

- 持っている間だけ速度を変える
- 離したら状態を戻す
- 持ったプレイヤーへメッセージを送る

使える値:

- `プレイヤー`

## メッセージを受け取ったとき

別のアイテムやプレイヤーからメッセージを受け取ったときに実行します。

使いどころ:

- 遠隔ドア
- 複数アイテム連動
- スイッチから別アイテムを動かす
- 受信値を変数に保存する

使える値:

- `受け取った数値`
- `受け取った文字`
- `受け取った真偽値`
- `送ってきた相手へ返信`

注意:

- メッセージ名が送信側と受信側で完全に一致している必要があります。

# パーツリファレンス

Jcreate++のパーツを、役割ごとにまとめたリファレンスです。

ブロック名は画面左のパーツ一覧に出てくる名前に近い形で書いています。細かい見た目は今後変わることがありますが、考え方は同じです。

## 読み方

- 何をするか: そのパーツの目的
- 置ける場所: どのイベントやブロックの中で使えるか
- Unity側の準備: Unity / Cluster側で必要な設定
- よくあるミス: 動かないときに見るところ

- [イベント](./events.md)

## Jcreate++の中で起きていること

Jcreate++は、ブロックを直接JavaScriptにしているわけではありません。

1. Blocklyのブロック配置を読む
2. Jcreate++内部のIRに変換する
3. IRからCluster Script用JavaScriptを生成する
4. 同時にUnity側で必要な設定を表示する

IRは「このブロックは音を鳴らす」「これは近くにプレイヤーがいる条件」のような意味を表す中間データです。これがあることで、ブロックの見た目と生成コードを分けて安全に扱えます。

## 共通の注意

- ほとんどの命令ブロックは、イベントブロックの中に置きます。
- 値ブロックは、数値や条件の穴に差し込んで使います。
- 受信値、衝突相手、プレイヤーなどは使えるイベントが限られます。
- Unity側の設定が必要なブロックは、生成コード欄の「Unity側で必要な設定」も確認してください。
